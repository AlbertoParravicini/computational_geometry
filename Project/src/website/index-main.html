Chapter 1


A gentle introduction to zonoids

Given a set of points \(S = \{p_1,\ p_2,\ \ldots,\ p_n\} \subset \mathbb{R}^d\), we all know what is the convex hull of \(S\), right?
In case you don't remember, we say that:

Convex Hull

$$CH(S) = \left \{ \sum_{i=1}^{n}{\lambda_ip_i:\ 0\leq\lambda_i\leq 1}, \sum_{i=1}^{n}{\lambda_i=1} \right \}$$
We can imagine to extend this definition by adding a constraint on the maximum value of \(\lambda_i\): given a real number \(k \in [1,\ \ldots, n]\), we define the k-zonoid of \(S\) as:

K-Zonoid

$$Z_k(S) = \left \{ \sum_{i=1}^{n}{\lambda_ip_i:\ 0\leq\lambda_i\leq\frac{1}{k}}, \sum_{i=1}^{n}{\lambda_i=1} \right \}$$
Note that the value of \(k\) doesn't necessarily have to be integer! However we'll see how restraining \(k\) to integer values will allow us to exploit interesting properties related to the geometrical structure of zonoids.

Also, unless otherwise specified, we'll assume that our points are bidimensional, i.e. belonging to \(\mathbb{R}^2\)). Working in 2 dimensions makes visualization much easier, but most of the properties of zonoids apply to any kind of dimensionality.

Even better, we'll see how zonoids are used in applications with extremely high dimensionality, and why their structure makes them suitable for such tasks!

What can we do with zonoids?

  • Compute the contour of the zonoid \(Z_k(S)\).
  • Compute the depth-map of \(S\), i.e. \(Z_1(S),\ \ldots,\ Z_n(S)\).
  • Check if a point \(x \in Z_k(S)\), for a given \(k\).
  • Test the depth of a point \(x\), i.e. the maximum \(k\) s.t. \(x \in Z_k(S)\).
But why would we want to do that? See Chapter 4 to know more!
A 3-zonoid of a set of points.

Interesting facts about zonoids

Immediately from the definition of zonoids, we can see some remarkable properties:

  • \(k\) can be a real or integer value, and also \(1\leq k \leq n\).
  • For \(k = 1\), we get the convex hull of our points, as \(0\leq\lambda_i\leq 1\), and so \(Z_1(S) = CH(S)\).
  • For \(k = n\), where \(|S| = n\), we find the mean point of \(S\), as \(\forall i\ \lambda_i = \frac{1}{n}\).
  • \(\forall k,\ Z_k(S)\) is a convex polygon.
  • If \(k_1 > k_2\), then \(Z_{k_1} \subseteq Z_{k_2}\).
  • From the previous 2 properties, \(Z_1(S),\ Z_2(S),\ \ldots,\ Z_n(S)\) is a sequence of nested convex polygons.

Zonoid depth

Given a point \(x\), and a set of points \(S\), we say that the zonoid depth of \(x\) with respect to \(S\) is the maximum value of \(k\) such that \(x \in Z_k(S)\). More formally $$D(x, S) = \left\{\begin{array}{ll} max\{k\ | \ x \in Z_k(S)\},\ if\ x \in CH(S) \\ 0,\ otherwise \end{array}\right.$$

Properties of zonoid depth

  • Translation invariant: \(D(x + b, S + b) = D(x, S),\ \forall b \in \mathbb{R}^d\).
    Informally, translating all our points by the same amount won't change the depth.
  • Linear invariant: \(D(Ax, AS) = D(x, S),\ \forall\) bijective linear transformations \(A:\mathbb{R}^d\ \rightarrow \mathbb{R}^d\).
    Rescaling our points by the same amount won't change the depth.
  • By combining the previous 2 properties, we can say that zonoid depth is invariant to affine transformations.
  • Zero at infinity: \(\lim_{\|x\| \rightarrow \infty}{D(x, S)} = 0\).
    If we take a point arbitrarily far away from \(S\), its depth will be \(0\). Actually, any point outside of \(CH(S)\) will have depth \(0\).
  • Upper semi-continuous: the set \(\{x \in \mathbb{R}^d | d(x, S) \geq \alpha\}\) is closed \(\forall \alpha \leq |S|\).
    In a sense, we can say that the region of space with depth \(\geq \alpha\) is finite.
  • Quasi-concave: \(D(\lambda x + (1-\lambda)y, S) \leq max\{D(x, S), D(y, S)\}, \forall \lambda \in [0, 1]\).
    If we take 2 points and draw a segment that joins them, the depth of any 2 points in the segment will be \(\leq\) than the maximum of the depth of the extreme points of the segment.

Chapter 2


Zonoids and K-Sets

The definition of zonoids that we have seen in Chapter 1 gives some insights about their properties, but it doesn't really tell much about how to compute the boundary of a zonoid, or its vertices!

In this chapter, it will be introduced a technique that allows us to easily draw zonoids, and how to apply it in practice by exploiting the so-called k-sets.

In this chapter:

  • How to find the vertices of a zonoid
  • K-sets
  • Exploiting k-sets to draw zonoids
  • An animated demo that shows how a zonoid is built from the k-sets!
Let's first think about how can we build a convex hull for a set of points \(S\): the vertices of the hull are the most extreme points of \(S\) in any given direction.
One can imagine of picking a direction and following it until it finds the last point of \(S\): that point will be a vertex of the convex hull; then repeat the process for all possible directions to get all the vertices!
Clearly, trying all directions isn't really feasible in practice, and we'll see later how to address the issue.

For now, let's try to generalize this approach so that it works with zonoids:
picking the most extreme point in a given direction can be seen as taking a linear combination of the points in \(S\), where the extreme point has a weight of \(\lambda = 1\), and all the others have weight \(\lambda = 0\) (note that, without loss of generality, the sum of the weight must be \(1\)).

But in a k-zonoid we can assign to each point a weight of at most \(\lambda = \frac{1}{k}\), so instead of giving a weight \(=1\) to the most extreme point, we can give a weight \(= \frac{1}{k}\) to the \(k\) most extreme points!

In other words, take the \(k\) most extreme points in a given direction, and compute their mean!

More formally, given a direction \(p\), the k-zonoid vertex \(x\) in that direction will be given by:
\(\operatorname*{arg\,max}_x \{p\cdot x\ |\ x \in Z_k(S)\} = \left ( \sum_{i=1}^{k}{\frac{1}{k}S_i^p} \right ) \)
Where \(S^p\) is the set of the most extreme point of \(S\) in direction \(p\). Basically we are taking a feasible point \(x \in Z_k(S)\) that optimize the gradient given by \(p\).

This approach works also for continuous values of \(k\), with a few modifications: in this case, assign the maximum allowed coefficient \(\frac{1}{k}\) to the most extreme \(\lfloor k \rfloor\) points, and the remaining \(1 - \frac{\lfloor k \rfloor}{k}\) to the next most extreme point in the given direction.

In the continuous case, a vertex \(x\) in a direction \(p\) will be given by

Zonoids with continuous k

$$\operatorname*{arg\,max}_x \{p\cdot x\ |\ x \in Z_k(S)\} = \left ( \sum_{i=1}^{\lfloor k \rfloor}{\frac{1}{k}S_i^p} \right ) + (1 - \frac{\lfloor k \rfloor}{k})S_{\lceil k \rceil}^p$$

K-Sets

The previous paragraph gave us an intuition of how zonoids can be built, but in practice we can't still do much!
Trying all possible directions to find the zonoid vertices is certainly out of the question, but what could we do instead?
We can exploit a geometric structure called k-sets.

K-Set

Given a set of points \(S\) in general position (no 3 points are collinear) and an integer \(k \in [1,\ n]\), a set \(S' \subseteq S\) is a k-set of \(S\) if \(|S'| = k\) and \(\exists\) a closed halfplane \(h\) such that \(h \cap S = S'\).
In 2 dimensions, it means that exists a straight line that separates \(S'\) can be from \(S\setminus S'\).
It turns out that exists a very strong relationship between zonoids and k-sets:

Lemma

Given a set of points \(S\) and an integer \(k \in [1,\ n]\), there is a bijection between the vertices of \(Z_k(S)\) and the k-sets of \(S\).

How can we construct the k-sets?


A very simple approach is to consider all pairs of points of \(S\), and for each pair, the line that crosses the 2 points.
Then we can check if \(k\) points lie on any of the 2 sides of the line: if so, we have found a k-set, and the line is the separator that defines the k-set.
Each 1-set contains only 1 point. The set of 1-sets will give the convex hull of \(S\).

The above approach won't work for \(k = n\) and \(k = n-1\), but we can easily overcome the issue:
  • For \(k=n\), we can simply take every point in \(S\), and get the unique k-set of size \(n\).
  • For \(k = n-1\), we can compute the k-set for \(k=n-3\), and then add to every k-set the 2 points that define the separating line.

Then, for each k-set, we compute its mean and find the zonoid vertices.

What about continuous k-sets?


We can compute the k-set for \(\lfloor k \rfloor\) and give to each point a weight of \(\frac{1}{k}\), then add to the k-set one of the points on the separating line, with a weight of \(1 - \frac{\lfloor k \rfloor}{k}\).

BUILDING ZONOIDS FROM K-SETS

On the left, we can see how each vertex of a zonoid is found from a single k-set.
  • We compute a k-set, then the mean of the points in the k-set will give a zonoid vertex.
  • Repeat the process for every zonoid to build the entire zonoid.
